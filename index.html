Seller Tool for Etsy

Toolshelper

Toolshelper is a browser extension for Etsy sellers that adds a compact insight card below each visible product or listing directly inside Etsy, helping sellers review important listing information faster and more clearly.

Toolshelper uses Etsy's official OAuth authorization flow. Users connect with their regular Etsy account and do not need to enter a password, API key, or shared secret into the extension.

Overview

Toolshelper is built to improve the listing review workflow for Etsy sellers by displaying useful listing insights directly where the seller is already working.

Instead of opening listings one by one, the seller can review key information immediately below each listing in a small, readable, minimal card.

The main goal of Toolshelper is to provide clear listing insights directly inside Etsy while keeping the original Etsy experience as intact and uncluttered as possible.

Core Listing Insights

Created Date

Shows when the listing was originally created.

Updated Date

Shows the most recent update date for the listing.

Favorites Count

Shows how many users have favorited the listing.

Sales Per Listing

Shows listing-level sales insights calculated from approved Etsy data.

How the Card Appears

  • The card appears directly below each product or listing inside Etsy.
  • The card is injected into the page DOM.
  • The design is intentionally compact, clean, and easy to scan.
  • The layout is designed to avoid disrupting Etsy's original page structure.
  • The card focuses only on the most useful listing information.

Sort by Newest

Toolshelper includes a Sort by Newest option to help sellers review newly created listings first.

  • The sorting target is based on listing creation date.
  • This supports a faster review workflow for recent listings.
  • The sorting behavior is intended to remain simple and predictable.

How the Product Works

Extension Responsibilities

  • Reads visible Etsy listings from the current page
  • Identifies each listing inside the page DOM
  • Requests listing insight data from the serverless backend
  • Inserts a compact insight card below each listing

Serverless Backend Responsibilities

  • Stores the application's API key securely
  • Handles Etsy OAuth authorization
  • Retrieves approved Etsy data through the Etsy API
  • Builds listing insights for the extension
  • Returns created date, updated date, favorites count, and sales per listing

Data Flow Overview

  1. The seller installs the Toolshelper browser extension.
  2. The seller connects their Etsy account using Etsy's official OAuth flow.
  3. The extension reads visible listings on the current Etsy page.
  4. The extension sends listing identifiers to the serverless backend.
  5. The backend retrieves approved Etsy data and prepares listing insights.
  6. The extension renders a card below each listing inside Etsy.

Important Sales Note

Sales per listing is not treated as a simple direct page field.

This value may require server-side calculation using approved Etsy transaction or order-related data associated with the specific listing.

For this reason, sales per listing is generated by the serverless backend and then returned to the extension for display.

How OAuth Works

Toolshelper uses Etsy's official OAuth authorization process to connect a seller account securely.

  1. The user clicks the connect action.
  2. The user is redirected to Etsy's official authorization page.
  3. The user signs in with their regular Etsy account.
  4. The user approves the requested access.
  5. The serverless backend completes the OAuth flow and stores the authorized connection securely.

Toolshelper does not ask the user to enter their Etsy password directly into the extension.

Permissions and Access

Toolshelper is intended to request only the permissions necessary for its core listing insight features.

  • Read access for approved listing-related data
  • Read access for approved seller account data needed for the feature
  • Read access for approved transaction or order-related information where needed to calculate sales per listing

Toolshelper is not intended to request unnecessary write access for editing, deleting, or changing shop content.

Read-Only Insight Experience

Toolshelper is designed to provide listing insights and improve seller review workflows.

It is not intended to create, edit, delete, or modify listings directly from the extension interface.

User Experience Goals

  • Compact, minimal card design
  • Fast review of important listing details
  • Clean display inside Etsy pages
  • Low visual clutter
  • Minimal disruption to Etsy's original interface

Performance and Stability Goals

  • The extension should work efficiently on pages with multiple listings.
  • Repeated requests should be reduced through server-side handling and caching where appropriate.
  • The same listing card should not be injected more than once.
  • The extension should continue working as new products appear on the page.
  • The card should load with clear behavior even when data is still being retrieved.

Frequently Asked Questions

Does Toolshelper ask for my Etsy password?

No. Users sign in through Etsy's official OAuth flow. Toolshelper does not ask users to enter their password directly into the extension.

Do users need to create their own Etsy API key?

No. Users connect with their Etsy account only. They do not need to create or paste their own Etsy API key or shared secret.

What information does the card display?

The card displays created date, updated date, favorites count, and sales per listing.

Does Toolshelper modify listings?

No. Toolshelper is intended to provide read-only listing insight features and not to modify shop content.

Why is sales per listing handled by the backend?

Because sales per listing may require calculation from approved Etsy transaction or order-related data rather than a simple direct field.
